#  一、好产品是增长的根本

在产品大规模推广之前，找到产品的核心价值，以及“啊哈时刻”。“啊哈时刻”是⽤户真正发现产品核⼼价值，他们为何需要它以及他们能从中得到什么的时刻。

通过产品不可或缺性调查（通过寻找活跃用户访问）、留存分析判断用户是否喜欢一个产品。

如何找到“啊哈时刻”？

- 客户调查，包括采访和实地调研，真正与客户和潜在客户交谈；
- 针对产品改动好信息传达进行高效试验；
- 分析用户数据：深挖用户行为数据，跟踪活跃用户的行为；

# 二、确定增长杠杆

>打造一个有吸引力的产品并通过不可或缺性测试是获得迅速、可持续增长的前提条件，但是只靠一个不可或缺的产品还不够。即便是真正出色的产品，即便它深受一群早期核心用户的喜爱，如果没有全力驱动增长的努力，也难免会失败。

如何明确增长战略？

第一，找到适合公司所处阶段的增长策略。

在增长初期，制定一个在最短时间内会对增长产生最大影响的试验策略。起步时关注点越集中，试验的目的性就越强，影响力就越大。

第二，明确哪些指标对产品增长来说事重要的。

不同类型的产品衡量的指标不一样，比如，“日活跃用户”对抖音来说是个重要指标，因为抖音的广告收入是以大量用户在APP上的停留时间为前提的。而对于滴滴打车来说，“日活跃用户”就不是他们最重要的指标，可能新用户注册量（用户和司机）才是。对于大众点评来说，被评价的商家数和每家商家评论数是核心增长因素。对于Facebook，关键因素是用户的分享数和用户花在查看更多动态消息的时间，新分享的内容会使用户花更多时间浏览，对于广告主更具有吸引力。

第三，明确北极星指标，且不要偏离路线。

在众多核心指标中找到北极星指标。比如，WhatsApp或者Wechat这类免费通信应用，北极星指标是用户发送的信息数，衡量这款App是不是用户和他的亲朋好友联络的主要方式。Airbnb的北极星指标是客房的预定量。

第四，指标因时而变。注意调整。

# 三、快节奏试验

> 增长黑客的巨大成功往往来自一连串小成功的累加。量变带来质变。一点一滴的认知学习都会带来更好的表现，催生出更好的试验想法，进而带来更多 成功，最终将一次次小幅的改善转化为压倒性的竞争优势

增长黑客的过程可以分为几个阶段：**分析数据并收集洞察，形成试验想法，排定试验优先级，运行试验，再回到分析阶段审视试验结果并决定下一步行动。**

## 第一阶段：分析

分析用户画像，用户行为特征，找到有明显特征的用户群体。

我们的最佳客户有哪些行为：

- 他们使用了哪些功能？他们访问了App的哪几屏？他们打开App的频率如何？
- 他们购买了哪些商品？他们的平均订单金额是多少？他们在一天中的什么时间下单？通常在哪些日子下单？

我们的最佳客户有哪些特征：

- 他们是从什么渠道转化为我们的客户的？是广告、推广邮件还是其他渠道？
- 他们使用什么设备？
- 他们具有哪些人口学特征，如年龄、收入等？他们居住在哪些地区？
- 他们距离最近的本品牌商店或其他商店有多远？
- 他们还使用其他哪些类似的App？

导致用户弃用App的原因：

- 哪些屏的退出率最高？
- App存在哪些阻碍用户采取某一行动的程序错误？
- 与其他提供商相比本应用中的商品价格如何？消费用户的行为中有哪些是弃用用户没有的？
- 他们从开始到弃用App的过程是怎样的？在弃用之前他们在App中花费了多少时间？

## 第二阶段：提出想法

想法应当按照一个事先制定的模板提交到“储备库”中。

**想法名称**：给想法去一个名字。

**想法描述**：“谁”（想法针对谁）“什么”（要创造什么？一个文案还是功能）“何处”（新功能活新文案在何处执行，是主屏还是其他地方）“何时”（何时出现在用户屏幕上）“为什么”（想法背后论证的过程）“如何”（建议开展的测试类型，A/B测试还是开发新功能）

**假设**：根据历史数据测算，或者类似实验，相关经验进行预估结果，能够让团队了解到一个想法可能带来的量化结果。

**待测指标**：明确评估测试结果需要追踪哪些指标。

## 第三阶段：排定优先级

“储备库”中想法可以使用ICE评分体系进行排定优先级。

- Impact（影响力）：想法对于团队关注的指标预期提升的程度。
- Confidence（信心）：对想法产生预期影响的信心。
- Easy（简易性）：进行一项试验所需投入的时间和资源。

## 第四阶段：测试

对提出的想法进行测验，注意测试规范性。

测试结束，需要对测试结果进行分析并总结学习，回到第一步形成一个循环。总结分析内容应该包括：

- 试验名称和描述，包括使用的变量和目标客户。

- 试验类型。测试的是产品功能、网页或App某屏上的营销文案的修改还是某个创意，抑或是新的营销策略？

- 受影响的特征。这可能包括试验在网站上或是App中运行位置的截图，或者某个广告牌、电视或电台广告中某个创意的副本。

- 关键指标。通过试验希望改进的指标是什么？

- 试验时间点，包括起止日期，也要说明当天是一周中的哪一天。

- 试验假设与结果，包括最初的ICE得分、样本量、置信水平和统计功效。

- 潜在干扰因素。比如试验运行的季节，或者是否有其他促销活动可能影响了访客行为。

- 结论。

# 四、获客：优化成本，扩大规模

在投入资金大规模获客之前，需要确定产品的增长杠杆且经过测试后确定了产品的”不可或缺“性。

## 扩大获客规模，要先实现两种匹配

### 语言-市场匹配，对产品优势的描述要能打动目标用户

**语言包括：广告语、产品本身包含信息、着陆页、每个功能、每个屏幕、每个页面的描述**

随着时代发展，用户的注意力持续时间非常短暂，因此在短时间内吸引用户的注意力，进一步点燃他们的需求和欲望非常重要。语言必须非常简洁的传达出产品的核心价值，且回答消费者最关心的问题：你展示的产品能满足我的什么需求。

**如何找到最佳广告语？**

**1）通过AB Test来测出最佳广告语；**

**2）通过一些渠道来寻找最佳文字和词组**

- 使用用户在社交网站或者网络评论中用来描述你的产品及其优点的语言；
- 利用前期产品“不可或缺性”调查中用户留下的评论；
- 直接给用户打电话，简单询问他们如何向自己的朋友或者同事介绍你的产品及其价值，这样往往也能找到非常好的用语和表述；
- 和客户服务团队交流；
- 查看用户来电记录、浏览论坛和在线产品评论来了解目标用户使用的语言；

有时对语言进行细微的调整能够改善获客效果。

### 渠道-产品匹配，选择的营销渠道要对目标用户有效

通过两个步骤锁定最佳渠道：发现和优化。

发现阶段：先对各种渠道进行深入调研，选出若干优先项进行试验。

优化阶段：找到一两个高度匹配的渠道，可以再扩大规模的同时努力实现渠道成本效益比和影响力的最大化。

3种主要渠道类型：

|     病毒/口碑渠道      |   有机渠道   |         付费渠道         |
| :--------------------: | :----------: | :----------------------: |
| 社交媒体（微博、抖音） | 搜索引擎优化 |  线上广告（抖音、快手）  |
|      朋友推荐计划      | 应用商店优化 | 线下广告（电视、印刷品） |
|       比赛和礼品       |   免费工具   |           电台           |
|                        | 电子邮件营销 |           赞助           |
|                        |   内容营销   |                          |

缩小渠道选择范围的方法可以考虑**用户特征和习惯**。

|                    用户行为                    |   可以考虑的渠道   |
| :--------------------------------------------: | :----------------: |
|         是否使用搜索引擎寻找解决方案？         |      SEO/SEM       |
| 用户是否通过口口相传的方式和朋友分享你的产品？ | 病毒渠道或推荐计划 |
|        用户数量增加是否会改善用户体验？        |       病毒式       |
|      你的目标用户是否已经在使用别的平台？      |     整合与合作     |
|           用户是否有很高的终身价值？           |      付费获取      |

选出可测试的渠道，可以针对每种渠道提出一系列可用于试验的具体策略，并确定优先级。

确定优先渠道，可以在这几个方面打分，得到平均分在做辅助决定：

|   方面   |               解释               |
| :------: | :------------------------------: |
|   成本   |        试验的花费多少钱？        |
|   定向   |     是否容易接触到目标受众？     |
|   控制   |  试验一旦开始还能否修改，终止？  |
| 时间投入 |    团队启动试验需要多长时间？    |
| 产出时间 | 试验开始需要多久能拿到实验结果？ |
|   规模   |     试验覆盖人群规模有多大？     |

## 设计病毒循环

几个关键点：

- 要实现病毒式增长，产品必须提供真正价值

- 病毒式增长包括口碑式和原生病毒（内置于产品当中用来“捕获”更多用户的病毒机制）

  原生病毒让分享产品这种体验变得不可获取，便捷舒心。比如Hotmail的邮件签名只需要单击鼠标，完成一个非常简单的注册过程就能得到免费邮箱。

- K因子（客户发出的邀请数 * 受邀者中接受邀请的人数比列）不一定要超过1；病毒性可以使用以下公式来衡量：

  $$
  病毒性 = 有效载荷 \times 转化率 \times 频率
  $$
  
  有效载荷是指每位用户每次向多少人发送广告或者链接

在病毒营销当中要注意以下几点：

- 提供双向奖励，同时给邀请人和受邀人提供奖励。
- 挖缺产品的网络效应。网络效应是指使用产品的人数越多产品的体验越好，所以人们愿意鼓励其他人参加的）。比如微信支付，通过微信红包这个网络效应就增加了用户。
- 奖励机制与产品的核心价值契合。比如Dropbox的奖励机制为推荐给新好友可以获得存储空间。
- 确保受邀者获得满意的体验。让他们知道这个应用/网站是做什么的，能满足他们什么需求。

成功的案列是通过大量的试验得出的。

# 五、激活

**两大策略：优化新用户体验 + 让用户尽可能快的体验到产品的核心价值**

发现提高产品激活率思路和洞察的步骤：

1. 明确让用户体验到“啊哈时刻”的所有步骤；
2. 创建一个显示每个步骤转化率的漏斗报告并按照访问渠道对用户进行分类；
3. 对哪些在流失率很高的环节依然继续使用产品的用户以及弃用产品的用户进行问卷调查和采访，找出流失原因。
4. 你可以基于这些信息设计出新的、具有很强针对性并且效果显著的试验，以改善激活。

影响激活率的因素叫做激活障碍，在用户体验中会存在摩擦，摩擦的概念用户在使用过程中遇到的麻烦可能会使用户放弃这个产品。摩擦、欲望和转化的公式：欲望 - 摩擦 = 转化；可以通过增强欲望，也可以减少摩擦来提高整体转化。

新用户体验是最大的摩擦点。

## 优化新用户体验

设计一种独特的体验来吸引用户一直使用产品并看到产品的价值。通过创造独立页面来吸引新用户，这样做可以使增长团队更容易测验用户体验，也不用担心试验干扰到现有用户的体验。新用户体验的第一个着陆页必须完成三个根本任务：

- 传达相关性：着陆页与访客目的和欲望之间的匹配度；
- 展示产品价值：迅速并简洁地回答访客的疑惑：用户可以从中得到什么？
- 行为召唤：为访客提供一个极富吸引力的下一步行动。

两个广泛适用于各种产品和业务的策略：

1. 单点登录：即一键登录，点击按钮快速创建账户，极大改善转化；
2. 翻转漏斗：在邀请访客注册前先让他们体验产品带来的乐趣，提前感受到“啊哈时刻”。

## 优化与阻力的角力

在提升新用户激活率时，你可能需要靠增加阻力来引导用户进入后续环节。一方面要引导用户完成整个过程并最终实现激活，另一方面又不能给用户带来过多的负担，令他们望而却步。

行为召唤有很多方法：配上更多文字信息，插入评论，说明，还有注册按钮的文案与颜色。

## 积极的摩擦

并非所有的摩擦都会带来负面的影响。摩擦会增加用户投入产品的时间，但是积极地摩擦会让用户对产品的投入增加，从而他们对产品的忠诚度就越高。

尽管填写信息的提示可能是一个摩擦源，但如果操作得当——用奖励的方式通过行动逐渐提高用户承诺——摩擦也可以成为激活与增长的催化剂。

### 创造学习流

新用户体验设计利用了新用户在初次使用产品时给予你的大量关注和耐心，他能确保用户在体验结束时就可以被激活。

对于微博、抖音、知乎或者音乐类app，在新用户注册是，就会要求用户选择自己的喜爱的内容，然后给用户推对应的内容。但是像电商类app，用户的目的性很强烈，浏览商品后购物，所以不需要创造这样的学习流。

### 游戏化机制

游戏化能够带来挑战和乐趣，再加上提供有意义的奖励，那他可能成为一个强大的激活手段。

但是，如果奖励没有任何价值，或者奖励与用户体验的核心价值毫不相关（也就是没有实现“产品——奖励匹配”），就会让人觉得这是在捉弄用户或是让人觉得莫名其妙。

游戏化的三个主要方面：

- 有意义的奖励：地位、权限、权力、实物
- 通过改变赢取和展示奖励的方法来创造惊喜和乐趣
- 提供能带来即刻满足感的要素

### 触发物

触发物是指任何刺激人们采取行动的提示。常见的触发物有邮件通知、移动推送、站内信、Push、着陆页上的行为召唤。

使用触发物有一个重要的经验：接触用户的动机应该是提醒用户存在一个对他们明显有价值的机会。

比如：淘宝京东会通过推送告知用户在购物车或者收藏中的商品正在降价，或者有优惠的活动，这类的触发对用户来说事有价值的。一些可用于试验的常见触发物：

- 创建账户——鼓励下载App或者访问零售网站的用户创建账户。
- 购买通知——鼓励用户利用短期折扣购买商品。

- 重新激活行动——鼓励那些有一段时间没有访问网站或App的用户再次回来。
- 新功能通知——分享产品升级的消息。
- 顶级用户奖励——让那些大量使用产品的用户知道他们很特别，鼓励他们更多地使用产品，和产品建立更加紧密的联系。
- 行动或者状态改变——如朋友采取了一个行动或者购物车中某件商品的价格发生了变化。

设计触发物时，有6个说服原则：

- 互惠：人们更有可能因为礼尚往来去做一些事情，无论之前别人给你提供了何种帮助，而你现在需要给予别人某种帮助；
- 承诺和一致性：已经采取行动的人更可能再次采取行动，无论行动的大小或者内容发生了什么变化；
- 社会认同：当感到不确定性时，人们会先看到别人怎么做，然后自己才做决定；
- 权威：人们会参考权威人士的做法来决定自己采取什么行动；
- 好感：人们更愿意和自己喜欢的公司或者人做生意；
- 稀缺：当人们担心自己会错失良机时就会采取行动；

结论：一定要试验触发物。

# 六、留存

互联网目前已经进入下半场，拉新成本变得越来越贵，所以留存显得非常重要。留住用户时间越长，从他们身上获得更多收益的机会就越大，每位用户带来的平均收益增加后，便可以将更多资金用于增长投资，从而形成一个良性循环。

高留存率也可以让口碑营销或者病毒营销具有更好的效果。

**留住用户的根本**：提供可以持续满足用户需求或者令他们感到愉悦的优质产品或者服务，让产品或服务对他们而言不可或缺。

**出现用户流失的情况**

1）内部：产品不能满足用户新需求；

 2）外部：推出新产品或竞争对手推出新功能

**用户对产品的依赖性越来越高的情况**

1）产品具有“储存价值”，即用户在产品中存储的内容越多，用户越离不开这个产品

2）改进产品或服务

## 留存的三个阶段

1. 初期：激活阶段的延伸。让用户在特定时间段内重复使用产品并从中获得新体验达到一定的次数才能强化他们对产品价值的认识；
2. 中期：让使用产品成为一种习惯；
3. 长期：让产品继续为用户带来更大的价值，试验各种方法来不断完善产品

### 确定留存指标并跟踪群组（初期）

不同产品的留存指标是不同的，在设计留存指标时需要将计算结果与市场调研出的同类产品的最高留存指标进行对比。

确定留存指标后，可以利用群组分析来确定不同用户群的留存率。群组划分可以按照获客时间/ 第一次购买或注册的时间。

分析群组数据，找到各个节点流失的原因，制定破解方法。

### 让用户养成使用习惯（中期）

让用户形成习惯的关键在于让他们确信继续使用产品能给他们带来持续的回报。所以提升回报在用户眼中的价值可以带来更大的留存。

比如叮咚买菜的会员就提供了免运费+免费送菜+会员价格等这些回报，让用户感受到购买他的会员计划会觉得很值得。还有一个例子是淘宝的88会员，开通88会员的意义不仅仅是88元，而是改变人们的心理，让他们不去别的地方购物。

**提供实际的、体验式的回报**

除了提供优惠券、折扣或者礼物，还有三个需要关注的回报策略：

- **品牌大使计划：**结合社会回报（社会认可，高级称号）+实际回报（优惠，差异化服务体验）
- **认可用户成就：**当用户完成一个里程碑事件时，给他们发送祝贺邮件、推送。比如你的博文在CSDN平台上被点赞转发很多次，CSDN会给你发送邮件提示你祝贺你。从而用户会登陆到CSDN平台查看，并激发用户创作下一篇博文。
- **客户关系个性化：**大数据可以根据不同用户的特性为他们提供不同的服务。实现“千人千面”。

### 保持长期活跃（长期）

保持长期活跃的策略：**定期推出新功能 + 优化现有产品功能、推送以及对重复使用的奖励**

长期留存的一个关键要素是持续对用户进行引导，让他们了解之前没有发现的功能，感受产品最大价值。

## 召回策略

复活流失用户的策略主要是向他们发送短信、站内信、邮件以及定向广告来完成。

其中这些工具中的内容是需要根据数据分析出来的信息来提取。针对不同的人群发送不同的内容，有一些比较常见的分析方向：

- 是否在app中有社交朋友，朋友发送的内容；
- 是否在app中购物，折扣提醒，上新提醒；
- 近期热点讨论
- 关键节点、日期，比如生日，第一次注册时间
- 启动app的动机，比如因为周杰伦点开了qq音乐的用户，可以向他们发送周杰伦的新动态或者歌曲推送；

# 七、变现

不同类型的公司变现方式是不一样的。比如，电商类的变现就是争取让更多的用户下单付费；SaaS类的让更多订购者续订、升级服务；销售广告类的创收本质是要创造更多的广告位，并说服更过广告主出更高的价格来购买广告位。典型的就是字节跳动将流量变现，抖音是一个具有比较大流量的平台，某个用户在还没滑到广告的时候，抖音告诉所有广告主：“我马上要给这个人放广告了，你们出价吧，价高者得”，然后这个时候，所有广告主都和抖音联合建模，Pdd看了看模型分，这个用户是我的潜在用户，然后出价，价格最高竞价成功，然后抖音展示了Pdd的广告。

## 绘制变现漏斗

第一步：分析数据，绘制用户旅程路线图。从获客到留存的整个过程中找到所有可能从用户身上盈利的机会。以及阻碍创收的所有环节。

第二步：分析哪些环节带来的收益最高，哪些环节是损失潜在收益的地方（夹点）。对于夹点，对他们进行针对性试验。

## 关注每个群组的贡献

每个群组的变现能力是不一样的，这就可以对他们做不同的策略。（根据用户画像来做差异化策略）

划分群组的维度也比较多：

1. 根据**用户创造的收益**的高低来建立群组：电商类公司，根据每年（每季度或者每月或者每周）的消费额来划分；订阅服务类按照订购计划的档位来划分群组；广告盈利的公司按照用户参与度（页面逗留时间，浏览数量等）+ 变现指标（从每位用户身上获取的收入）
2. 根据**用户属性**：性别，年龄，学历，地理位置等
3. 根据**用户行为**：启动app次数，点击行为等

## 优化定价

产品的定价需要与产品、财务团队合作，通过开展问卷调查和用户调研找出最佳定价区间并进行试验。

首先在用户中开展问卷调查，询问受访者哪些功能对他们而言最重要以及愿意为此支付什么样的价格。结合各项成本以及竞争产品的调研，与财务团队共同来确定价格。

**价值度量**：决定用户付费额的度量。衡量指标与用户从服务中获得的价值直接挂钩。

**心动价格**：以9、8、99、98、95结尾的价格对消费者来说更具有吸引力。

确定好定价后，如何在价格页面以及其他地方向用户呈现并传达这些价格信息同等重要。

## 定价相对论

人们对于价格高低的判断会受到其他价格选项的影响。可以使用烟雾弹产品，驱使用户购买更高价位的产品。

关于烟雾弹产品：例如马卡龙app提供年会员和月会员，可以增加一个新的会员计划，使它在功能上略逊于年会员，价格也略便宜，这样能够利用价格相对论现象驱使更多人购买年计划。团队应该试验增加一个产品选项，利用定价来帮助用户更好地了解你向他们出售的产品和计划的相对价值。

有些时候，低价并不一定能带来更高的销量，有时提高价格可能会提高转化、利润，可以对此进行试验。

## 六大原则

六个原则来寻找实现变现的增长手段：

### 互惠原则

不要急着从用户身上变现，可以先让用户体验一些免费的功能后，再从中寻找变现的方法。

### 承诺和一致原则

一旦我们采取某个行动，我们就有可能继续采取与之前相一致的新行动.比如心愿书单，用户可以将想买的书加入到书单中，这个操作相当于用户已经做了承诺，那后续用户购买这本书的概率就会比较大。

### 社会认同原则

增加证词、指明客户的企业标志、客户使用产品的结果、当前在你的网站上浏览或者购物的人数等，这些都可以向用户提供已经采取相同行动的其他客户信息，从而让潜在用户相信他们的决定是对的。

### 权威原则

相比普通人，我们更愿意相信专家和权威人士的话。

### 喜好原则

如果深受我们喜欢的人推荐了一款产品或者服务的话，我们会买的更多。

### 稀缺原则

当我们觉得自己要错失良机的时候，我们采取行动的可能性更大。比如电商app限定促销时间
